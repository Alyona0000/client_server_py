- [x] Зробіть два Python-скрипти: quiz_server.py і quiz_client.py, які працюють по TCP (IPv4,
SOCK_STREAM). Сервер і клієнт мають запускатися на одному комп’ютері (localhost) і
використовувати фіксований порт, який ви задаєте в коді.



- [x] Обмін повідомленнями між клієнтом і сервером має бути текстовим. Кожне
повідомлення — це один рядок, який обов’язково закінчується символом переводу
рядка \n. Клієнт відправляє команди рядками, сервер відповідає рядками.



- [ ] Реалізуйте на сервері квіз (тест) з мінімум 5 питаннями. Для кожного питання повинні
бути: текст питання, чотири варіанти відповіді A/B/C/D, і одна правильна відповідь.
Питання можна зберігати прямо в коді сервера.




Сервер має підтримувати одну сесію для одного клієнта: після підключення клієнта
сервер веде з ним діалог, пам’ятає ім’я користувача, поточне питання, чи була дана
відповідь на поточне питання, і поточний рахунок. Мінімально достатньо, щоб сервер
обслуговував одного клієнта за раз (інші підключення можуть чекати або не
підтримуватись).





Клієнт повинен дозволяти користувачу в консолі вводити команди та відправляти їх на
сервер. Після кожної відправленої команди клієнт повинен отримати відповідь сервера
і вивести її на екран. Якщо сервер повертає питання, клієнт має виводити його у
зрозумілому вигляді (питання та варіанти A/B/C/D).
Клієнт надсилає на сервер команди у вигляді рядків строго в одному з наступних
форматів: LOGIN <name>, START, Q, ANSWER <A|B|C|D>, NEXT, SCORE, HELP, QUIT. В
командах, де є <...>, значення передаються через пробіл після назви команди.
Сервер відповідає рядками у форматі одного з наступних типів: OK <message> (успішне
виконання), ERR <reason> (помилка), Q <id>;<question text>;<A>;<B>;<C>;<D>
(повернення поточного питання), END <final_score>/<total> (завершення квізу). Формат
відповіді Q ... повинен використовувати символ ; як роздільник полів.
До виконання команди LOGIN сервер не повинен приймати жодні інші команди: на
будь-яку іншу команду сервер має відповідати ERR you must LOGIN first. Після успішного
LOGIN, але до виконання START, сервер має дозволяти тільки START, HELP і QUIT, а на Q,
ANSWER, NEXT, SCORE відповідати ERR quiz not started.
Після виконання START сервер має дозволяти проходження квізу. Команда Q повинна
повертати поточне питання у форматі Q .... Команда ANSWER <A|B|C|D> повинна
приймати відповідь на поточне питання і повертати OK correct або OK wrong. На одне
питання не можна відповідати двічі: повторна спроба відповісти на те саме питання
повинна давати ERR already answered. Команда NEXT повинна переводити на наступне
питання тільки якщо на поточне питання вже була дана відповідь, інакше повинна
повертати ERR answer first. Команда SCORE повинна повертати поточний рахунок у

повідомленні OK ... (формат повідомлення ви визначаєте самі, але це має бути
однозначний текст з рахунком).
Коли питання закінчились, сервер повинен надіслати END <final_score>/<total> і
закрити з’єднання з клієнтом. Команда QUIT повинна коректно завершувати сесію:
сервер має надіслати повідомлення типу OK ... і закрити з’єднання.
Сервер не повинен завершувати роботу з помилкою через некоректні команди клієнта.
Для невідомих команд сервер має відповідати ERR unknown command. Для
неправильних аргументів (наприклад, відсутнє ім’я в LOGIN, неправильна літера в
ANSWER, зайві аргументи там, де їх не повинно бути) сервер має відповідати ERR invalid
arguments.
